Prune commit created. Metadata attached to ephemeral targets like commit SHAs and path entries can accumulate over time and bloat the metadata ref. A prune commit removes these scopes from the tree, keeping only durable metadata (branch and project). Because the pruned data is still reachable in earlier commits, it can be recovered from git history if needed — but the current tree stays lean. This is the equivalent of garbage collecting metadata that is no longer relevant to the active state of the repository.
